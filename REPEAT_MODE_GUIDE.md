# StressAgent 고도화 - 반복 모드 가이드

## 개요

StressAgent가 고도화되었습니다! 이제 두 가지 모드를 지원합니다:
- **모드 1: 단순 작업 모드** - 기존 방식 (1회 실행, 결과 저장)
- **모드 2: 반복 모드** - 새로운 기능 (연속 실행, 워커 수 랜덤, 결과 미저장)

---

## 실행 방법

### 1️⃣ 기본 실행 (모드 선택 UI)
```bash
node bin/stressagent.js config-wellerp.yml
```

실행 시 모드 선택 메뉴가 표시됩니다:
```
실행 모드를 선택하세요:
  1) 단순 작업 모드 - config 설정대로 1회 실행, 결과 저장
  2) 반복 모드     - 연속 반복 실행, 워커 수 랜덤, 결과 미저장

선택 (1 또는 2, 기본값: 1): 
```

### 2️⃣ 단순 모드 직접 실행
```bash
node bin/stressagent.js config-wellerp.yml -s
# 또는
node bin/stressagent.js config-wellerp.yml --simple
```

### 3️⃣ 반복 모드 직접 실행
```bash
node bin/stressagent.js config-wellerp.yml -r
# 또는
node bin/stressagent.js config-wellerp.yml --repeat
```

## 모드별 상세 설명

### 모드 1: 단순 작업 모드

- config 파일 설정대로 정확하게 1회 실행

- 최종 리포트 출력
- JSON + CSV 결과 자동 저장

### 모드 2: 반복 모드 (신규)

**특징:**
- ✅ 무한 반복 실행
- ✅ 각 반복마다 워커 수를 1 ~ config.workers 범위 내에서 **랜덤 할당**
- ✅ 모든 워커 완료 → 1초 대기 → 다음 반복
- ✅ 간단한 요약 정보만 출력 (보고서 미저장)
- ✅ 하루 종일 부하를 계속 유발 가능

## 반복 모드 사용 예시

### 예시 1: WELLERP 로그인 시나리오로 하루 종일 부하 테스트
```bash
# 워커 1~5개 범위에서 랜덤 할당, 각 반복마다 로그인 + 페이지 순회
node bin/stressagent.js config-wellerp.yml -r
```

**결과:**
- 각 task마다 랜덤한 수의 동시 로그인 시도
- 약 30초마다 다음 task 시작 (작업 시간 + 1초 대기)
- 24시간 연속 실행 가능
- CPU/메모리 등 시스템 리소스를 부하에 따라 자동 조절

### 예시 2: 간단한 페이지 접속 테스트
```bash
# 워커 1~3개 범위에서 랜덤 할당
node bin/stressagent.js config-simple-access.yml -r
```

**결과:**
- 약 3초마다 다음 task 시작
- 빠른 반복으로 높은 요청 볼륨 발생
- 서버의 동시 연결 처리 능력 테스트

## 설정 파일 구성

반복 모드에서 `workers` 값의 의미:

```yaml
# config-wellerp.yml (반복 모드 사용 예정)
url: "http://121.136.244.39:18443/WELLERP/"
workers: 5                    # ← 최대 워커 수 (1~5 범위에서 랜덤)
iterations: 1                 # ← 각 워커의 반복 횟수 (변경 안 됨)
scenarioFile: "./scenarios/login-traverse.js"

# 반복 모드에서의 동작:
# Task #1: 워커 2개 × 1회 = 2개 요청
# Task #2: 워커 4개 × 1회 = 4개 요청
# Task #3: 워커 1개 × 1회 = 1개 요청
# ...계속 반복
```

## 고급 사용법

### 1️⃣ 백그라운드에서 24시간 실행
```bash
# nohup으로 실행 (터미널 종료 후에도 계속 실행)
nohup node bin/stressagent.js config-wellerp.yml -r > stress-test.log 2>&1 &

# 진행 상황 확인
tail -f stress-test.log

# 중지
pkill -f "node bin/stressagent.js"
```

### 2️⃣ 여러 시나리오 병렬 실행
```bash
# 터미널 1: 로그인 시나리오
node bin/stressagent.js config-wellerp.yml -r

# 터미널 2: 간단한 접속 시나리오
node bin/stressagent.js config-simple-access.yml -r
```

### 3️⃣ 시스템 리소스 모니터링
```bash
# 병렬 실행하면서 리소스 모니터링
watch -n 1 'free -h && echo "---" && ps aux | grep node | grep -v grep'
```

## 문제 해결

### 문제 1: 반복 모드가 자동으로 종료됨
**원인**: 에러 발생 또는 리소스 부족

**해결책**:
```bash
# 에러 로그 확인
tail -f stress-test.log | grep "❌ 오류"

# 워커 수 감소
workers: 2  # 또는 더 낮은 값
```

### 문제 2: 메모리/CPU 급증
**원인**: 워커 수가 너무 많음

**해결책**:
```yaml
workers: 5  # 줄이기
slowMo: 100  # 입력 지연 증가
```

### 문제 3: 각 task 간격이 너무 길어짐
**원인**: 시나리오 실행 시간이 김

**해결책**:
- 더 간단한 시나리오 사용
- iterations 값 감소
- 로그인 대기 시간 단축
